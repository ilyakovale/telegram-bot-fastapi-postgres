# Telegram order bot

Telegram-бот для регистрации заказов и их администрирования.  
Проект состоит из 4 микросервисов: Gateway, Account, Order, Admin.  

> ⚠️ **Проект ещё не закончен. Готово примерно 60%.**  
> Работает основной функционал, часть админ панели и обработка заказов будут доделаны позже.

## Что умеет (уже сейчас)

- Принимать заказы от пользователей через Telegram  
- Сохранять профили с контактными данными  
- Администраторам – смотреть все заказы, менять статус, блокировать пользователей 

## Стек

- **Python 3.11+**  
- **aiogram 3** – работа с Telegram  
- **FastAPI** – общение между сервисами  
- **SQLAlchemy (async) + asyncpg** – база данных  
- **PostgreSQL**  
- **Pydantic v2** – валидация  
- **Docker, Docker Compose** – контейнеризация микросервисов

## Архитектура (коротко)

4 микросервиса:
- **Gateway** – бот, принимает все сообщения, отправляет запросы в другие сервисы  
- **Account** – пользовательские аккаунты  
- **Order** – работа с заказами  
- **Admin** – скрытая админ-панель  

Все общаются по HTTP внутри Docker-сети.

## Как запустить

1. Клонировать репозиторий:  
   `git clone https://github.com/ilyakovale/telegram-bot-fastapi-postgres.git`

2. Переименовать все `.env.example` в `.env` и заполнить переменные окружения.

3. Запустить:  
   `docker-compose up --build`

Бот готов к работе.

## Использование

- Обычный пользователь: жмёт /start, заполняет профиль, создаёт заказы через кнопки.  
- Администратор: ID должен быть в `.env`. После старта появляется кнопка «Администрирование» с доступом ко всем заказам и пользователям.
